Act 3 — The agent

The knowledge base powers an agent that
answers queries before support even sees them.

When a new Jira ticket or Teams query arrives, the agent searches the knowledge base and posts an answer automatically — if it's confident enough.

Step 01
📬
New ticket arrives
Developer raises a ticket in Jira or asks a question in a Teams channel. The agent is watching in real time via webhook.
→
Step 02
🔎
Knowledge base search
The query is embedded and compared against thousands of resolved tickets using semantic similarity — matching meaning, not just keywords.
Similarity threshold
Only responds when match confidence > 75%. Below that — stays silent and lets the support team handle it.
→
Step 03
✍️
AI generates answer
The LLM writes a concise, grounded answer using only the retrieved tickets — never hallucinating. Sources are cited by ticket ID.
→
Step 04
💬
Answer posted instantly
The answer is posted as a comment on the Jira ticket or Teams thread — labelled as an AI suggestion, with the source tickets listed.
🤝
For the support engineer: The agent is your assistant — not your replacement. It handles the repeat questions so you can focus on the genuinely hard ones.
